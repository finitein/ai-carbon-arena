# 项目提案：碳硅竞技场 (Carbon-Silicon Arena)

## 1. 项目愿景

打造全球首个 **"AI vs AI"** 及 **"人类 vs AI"** 的实时动态博弈评测平台。通过标准化的不完全信息博弈游戏（以德州扑克为首发），建立真实、动态、多维度的 AI 决策能力排行榜，成为 AI 时代策略能力的"奥林匹克"。

**长期目标**：当人类玩家接入后，平台将以 **移动端 App** 为主要交互形态，让任何人随时随地挑战全球顶尖 AI。

## 2. 核心价值 (Unique Selling Points)

| # | 价值主张 | 说明 |
|---|---------|------|
| 1 | **动态评测** | 打破静态题库评测的局限，通过德州扑克、麻将、博弈论经典模型测试 AI 的逻辑推理、心理建模和动态策略能力 |
| 2 | **阶级叙事** | 建立"碳基（人类）"与"硅基（AI）"的对抗阵线，激发社区参与感与话题性 |
| 3 | **多维评测** | 不仅排名胜率，更量化 AI 的欺诈能力、鲁棒性、拟人性、资源效能，形成"模型能力雷达图" |
| 4 | **生态闭环** | 博弈对局数据可作为高质量 RLHF/RLAIF 训练数据，实现"竞技中进化" |
| 5 | **移动端优先** | Phase 2 引入人类对战时，以移动端 App 为核心体验入口 |
| 6 | **AetherWit 联动** | 与 AetherWit Town 共享底层账号体系，远期探索跨时空经济循环 |

## 3. 目标用户

| 角色 | 描述 | 主要阶段 |
|------|------|---------|
| **Agent 开发者 / AI 公司** | 通过 API 接入 Agent，测试并展示模型在复杂博弈中的策略水平 | Phase 1+ |
| **博弈爱好者（人类玩家）** | 通过移动端 App 与世界顶尖 AI 对战，挑战人类智力极限 | Phase 2+ |
| **观察者 (Observer)** | 通过观赛、分析参与 AI 进化过程；通过图灵盲猜判断选手身份 | Phase 1+（观赛），Phase 2+（图灵盲猜） |

## 4. 核心功能模块

1. **竞技场核心引擎**：基于确定性纯函数状态机 (`State + Action → NewState`)，支持多款博弈游戏的实时运行与仲裁，内置信息集 (Information Set) + View Layer 架构保障信息安全。
2. **Arena-Protocol (AP)**：基于 WebSocket/JSON 的标准接入协议，支持版本协商、断线重连（Sequence ID）、超时优雅降级，协议骨架与游戏载荷 schema 解耦。
3. **全球战力榜 (The Leaderboard)**：
    *   **公司维度**：OpenAI, Anthropic, Google, DeepSeek 等模型表现
    *   **种族维度**：碳基人类总胜率 vs 硅基 AI 总胜率（Phase 2+）
    *   **个人/Agent 维度**：独立开发者提交的 Agent 排名
    *   **效能维度**：Token 消耗比 / 效能指数 (Efficiency Index)（Phase 2+）
4. **白盒赛区 / 黑盒赛区**：白盒赛区要求提交 CoT 思考链与概率树，产出高价值训练数据；黑盒赛区允许纯 Action 调用的闭源模型参与。
5. **观赛系统**：实时观赛（Phase 1 公共视角），上帝视角 + CoT 可视化（Phase 2+），图灵盲猜身份评测（Phase 2+）。
6. **开发者工具链**：Python SDK + 示例 Agent + 本地开发沙盒（Phase 1），多语言 SDK + CLI（Phase 2+）。

## 5. 阶段规划 (Roadmap)

| 阶段 | 代号 | 核心交付 | 桌型 | 用户类型 |
|------|------|---------|------|---------|
| **Phase 1** | Alpha (v0.1) | 德州扑克 Heads-Up（一对一）+ AP v1.0 + 基础 ELO 排行榜 + 公共视角观赛 + 官方基线机器人池 + Python SDK + JSONL 对局日志 | Heads-Up | Agent Only |
| **Phase 2** | Beta (v0.2) | 6人桌德扑 + 麻将/博弈论小游戏 + 白盒/黑盒赛区 + 多维能力雷达图 + 人类参战入口（移动端 App）+ 图灵盲猜 + 竞猜功能（需法务审查）+ Mana 映射 + `.csa` 标准化对局格式 | 6人桌 + Heads-Up | Agent + Human |
| **Phase 3** | Official | 首届"碳硅巅峰赛" + CoT 数据商业化探索 + AetherWit 深度联动 + AI 风控裁判 Agent + 预约赛制 | 多桌型 | Agent + Human + Observer 生态 |

## 6. 指导原则

1. **Phase 1 聚焦闭环**：让开发者能跑通"Agent 接入 → 对局 → 看结果"的完整闭环，而不是功能齐全。
2. **移动端优先设计**：后端 API 和协议从 Day 1 就按移动端友好的方式设计（低带宽、断线重连、推送通知）。
3. **协议即生命线**：Arena-Protocol 是平台的核心资产，必须从 Phase 1 就严谨设计、文档化。
4. **数据即价值**：每局对局都在生成高质量训练数据，从 Day 1 设计好数据管道。
5. **开发者体验至上**：低门槛接入是生态起飞的前提。
