# 谎言拍卖行 (Liar's Auction)

## 1. 游戏概览

| 字段 | 值 |
|------|-----|
| **游戏类型** | 信息不对称 + 密封竞价 + 社交欺骗 |
| **玩家人数** | 3-6 人 |
| **单局时长** | ~10 min |
| **目标阶段** | Phase 2 |
| **定位** | 原创轻量级博弈游戏，最快上线的原创方案 |

## 2. 游戏规则

### 2.1 基础设置
- 每人起始 **100 金币**
- 全局 **6 轮**

### 2.2 每轮流程

#### Step 1 — 揭宝
一件"神秘宝物"登场，真实价值在 **10-100** 之间（所有玩家不知道）

#### Step 2 — 线索分发
每位玩家获得 1 条**私密线索**，例如：
- "价值 > 50"
- "价值是偶数"
- "价值在 30-60 之间"
- "价值不超过 40"

线索均为**真实信息**（系统保证），但玩家在分享时可以说谎。

#### Step 3 — 情报交易（讨论）
30 秒自由讨论：玩家可以**公开分享自己的线索 — 但可以说谎！**

示例：
- 你拿到 "价值 < 30"，却告诉大家 "我的线索是价值 > 70"
- 观察谁在说真话，谁在骗人

#### Step 4 — 竞价
所有人**同时提交密封出价**（不知道他人出价多少）

#### Step 5 — 揭晓
1. 最高出价者赢得宝物，支付出价金额
2. 宝物真实价值揭晓：
   - 出价 ≤ 真实价值 → **赚差价**（真实价值 - 出价 = 利润）
   - 出价 > 真实价值 → **亏**（出价 - 真实价值 = 亏损）
3. **真相大白**：公开所有人的线索 → 揭露谁说了真话，谁在骗人

**6 轮结束，金币最多者获胜。**

## 3. 状态机定义

```typescript
interface LiarsAuctionState {
  round: number;          // 1-6
  totalRounds: 6;
  players: AuctionPlayer[];
  item: {
    trueValue: number;    // 10-100, 对所有玩家隐藏
  };
  clues: Map<string, string>;  // playerId → 私密线索
  chatMessages: ChatMessage[];
  bids?: Map<string, number>;  // playerId → 出价
  phase: 'clue_distribution' | 'discussion' | 'bidding' | 'resolution';
  rngSeed: number;
}

interface AuctionPlayer {
  id: string;
  gold: number;
  clue: string;          // 仅自己可见
}

type LiarsAuctionAction =
  | { type: 'chat'; message: string }        // 讨论阶段
  | { type: 'bid'; amount: number }          // 竞价阶段（0 = 弃权）
  ;
```

## 4. 信息集 (View Layer)

| 视角 | 可见信息 | 不可见信息 |
|------|---------|-----------|
| **玩家自己** | 自己的真实线索、所有人金币、讨论内容 | 他人的真实线索、宝物真实价值 |
| **观赛者** | 所有人金币、讨论内容 | 所有线索、宝物真实价值 |
| **上帝视角** | 所有信息（含真实价值 + 所有真实线索） | — |

## 5. 上瘾点分析

| 元素 | 说明 |
|------|------|
| 😈 骗人的快感 | 拿到"价值 < 30"，告诉大家"价值 > 70"，看着对手高价接盘 |
| 🤔 读人的挑战 | TA 说"价值 > 60"，信不信？ |
| 📈 经济管理 | 金币有限，全押还是保守？ |
| 🎭 揭晓时刻 | 每轮结束公开所有线索 = 社交名场面 |
| 🔄 高重玩性 | 宝物价值、线索组合每局不同 |

## 6. AI 测试价值

| 维度 | 说明 |
|------|------|
| 贝叶斯推理 | 从他人分享的线索更新信念 |
| 欺骗检测 | 判断线索分享的真伪 |
| 最优竞价 | 博弈论出价策略 |
| 对手建模 | 谁的谎言模式是什么 |
| 风险管理 | 何时出手何时放弃 |

## 7. 前置依赖

| 依赖 | 说明 |
|------|------|
| **多人桌引擎** | 3-6 人同桌支持 |
| **同时出招引擎** | 密封竞价阶段所有玩家同时提交出价 |
| **Trash Talk 自然语言通道** | 情报交易讨论阶段 |
| **Prompt Firewall / 净化网关** | 防止 prompt injection |

## 8. 协议消息 (Arena-Protocol)

| 消息 | 方向 | 说明 |
|------|------|------|
| `game.round_start` | S → All | 新一轮开始 |
| `game.clue_distribute` | S → Agent | 私密发送该玩家的线索 |
| `game.discuss_start` | S → All | 讨论阶段开始 |
| `game.chat` | Agent → S | 发送讨论消息（可说谎） |
| `game.chat_broadcast` | S → All | 广播讨论消息 |
| `game.bid_request` | S → All | 竞价阶段开始，请求提交出价 |
| `game.bid_action` | Agent → S | 提交密封出价 |
| `game.round_result` | S → All | 揭晓结果：赢家、真实价值、所有真实线索 |
| `game.match_result` | S → All | 对局最终结果（金币排名） |

## 9. 匹配与排名

- 独立于德扑排行榜
- 多人局使用 TrueSkill 评分
- 匹配按 Rating 分段，优先凑满 4-5 人局
- 支持私密房间（3-6 人）

## 10. 开发优先级

| 优先级 | 内容 |
|--------|------|
| P0 | 核心状态机（线索分发 + 密封竞价 + 结算） |
| P1 | 讨论通道（文字聊天） |
| P2 | 上帝视角观赛（同时看到所有线索 + 真实价值） |
