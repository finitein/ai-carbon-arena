# 碳硅夺宝 (Heist Royale) v3

## 1. 游戏概览

| 字段 | 值 |
|------|-----|
| **游戏类型** | 合作/背叛 + 信息博弈 + 经济管理 |
| **玩家人数** | 4-6 人 |
| **单局时长** | ~14-17 min |
| **目标阶段** | Phase 2.5 ~ Phase 3 |
| **定位** | 原创深度社交博弈，最高重玩性 |
| **设计参考** | Among Us 信任博弈 + 自走棋经济系统 + Liar's Bar 紧张感 |

## 2. 核心设计哲学

> **没有免费的选择。** 贡献、背叛、指控、沉默 — 全部有风险有代价。

## 3. 基本参数

- **4-6 人**，**5 轮制**
- 每人起始 **50 金币** + **5 张手牌**
- 手牌构成：**1-2 张破坏卡** + **3-4 张战力卡**（随机分配）
- 全场破坏卡总数**公开**（如"本局共 8 张破坏卡流通"），个人持有数未知

## 4. 每轮流程

### Phase 1 — 战前讨论 (Pre-Commit) 📢
- **1 轮顺序发言**，**不可追加**
- 每人 15 秒发言
- 首发位置每轮顺时针轮转
- 目的：承诺/喊话/分工（"我出 +5，大家放心！"）
- 这些承诺**可以被打破** — 被打破的承诺创造戏剧性名场面

### Phase 2 — 暗投 (Commit) 🃏
- 系统公布任务目标（需 X 点总战力）
- 每人从手牌**暗出 1 张卡**：
  - 战力卡：+1 到 +5 点
  - 破坏卡：**-3 点**

### Phase 3 — 宣言讨论 (Declaration) 💬
- **1 轮基础顺序发言**（每人 20 秒）+ **最多追加 2 轮**（每人 15 秒）
- 每人宣言自己出了什么卡（可说真话/说谎）
- 追加发言由**任一玩家发起即触发**（防止多数压制少数）
- **透视卡在此阶段前生效**

> 两轮讨论功能不同：
> - 战前讨论 = "我们**应该**怎么做" → 建立承诺
> - 宣言讨论 = "我**做了**什么" → 验证/质疑/找骗子
> 
> 形成 **承诺 → 行动 → 验证** 的叙事弧线

### Phase 4 — 指控 (Accusation) ⚔️

**两个固定档位**：

| 档位 | 成本 | 指控正确 | 指控错误 |
|------|------|---------|---------|
| 🟡 轻微指控 | 5 金 | +10 金 (净赚 5) | -8 金 (净亏 3) |
| 🔴 强力指控 | 15 金 | +30 金 (净赚 15) | -23 金 (净亏 8) |

- 每人可指控 **1 名玩家**（选档位），或选择**沉默**
- **沉默税**：2 金币进公共奖池 — 沉默安全但不免费
- **嫁祸卡可在此阶段打出** → 将指控转移给另一人

### Phase 5 — 揭牌 & 结算 🎴

**揭牌规则：公开卡牌组合，但不公开谁出了哪张**

> 示例："+5, +3, +2, +1, **-3** → 净战力 8 → ❌ 任务失败"
> 所有人知道有 1 张破坏卡，但**不知道是谁出的**

**为什么不公开破坏者身份**：
- 跨轮悬疑链：信息积累，每轮推理延续到下一轮
- 透视卡成为唯一确认身份的手段（价值极高）
- 嫁祸卡有意义：能真正转移焦点
- 指控成为主动情报手段（付金币买确认信息）

**指控结算**：
- 系统**私密通知**指控者结果（正确/错误 + 金币变动）
- 他人只能看到"A 对 B 发起了强力指控"（公开），结果未知
- 金币总量每轮末可见，但具体交易来源不透明

**任务结算**：
- **成功**（总战力 ≥ 目标）→ 奖金由所有贡献者**平均分配**
- **失败** → 破坏者平分 60%；贡献者得安慰奖（战力点 × 1 金币）；40% 滚入下轮奖池

> 平均分配理由：让游戏聚焦于"贡献 vs 背叛"的社交博弈，而非"出多大牌"的数学优化。

### Phase 6 — 卡牌商店 🛒
- 每人从随机 3 张中购买 1 张补手牌
- 战力卡（+1~+5，价格递增）/ 破坏卡（便宜）/ 特殊卡

**特殊卡**：

| 卡牌 | 效果 | 价格 |
|------|------|------|
| 🔍 透视 | 偷看一名玩家本轮暗投的卡（指控前生效） | 高 |
| 🛡️ 嫁祸 | 被指控时打出，指控转移给另一人 | 中 |
| 💣 双面间谍 | 揭牌前决定充当 +2 战力或 -3 破坏 | 高 |

## 5. 发言机制

### 顺序发言制
所有讨论采用顺序发言（非自由讨论）：移动端文字聊天防混乱、AI-vs-Human 统一节奏

### 发言顺序
**轮转制**：每轮首发顺时针轮转，避免先后发优势固化

### 追加发言规则

| | 战前讨论 | 宣言讨论 |
|--|---------|---------|
| 基础轮 | 1 轮 | 1 轮 |
| 可追加 | ❌ 不可 | ✅ 最多 2 轮 |
| 触发方式 | — | 任一玩家发起即触发，只触发自己是否增加轮次，可选择不增加 |

### 时间控制

| 阶段 | 每人时长 | 5人局总时长 |
|------|---------|-----------|
| 战前讨论 (1轮) | 15s | ~75s |
| 宣言基础轮 | 20s | ~100s |
| 宣言追加轮 | 15s/轮 | ~75s/轮 |

提前提交 → 跳过等待，全员提前提交 → 立即进入下一环节

## 6. 风险/收益平衡

| 行为 | 风险 | 收益 |
|------|------|------|
| 出战力卡 | 被破坏导致任务失败，白投 | 任务成功时平均分金 |
| 出破坏卡 | 被指控 → 赔双倍赌注 | 任务失败分 60% |
| 轻微指控 | 指控错 → 净亏 3 金 | 指控对 → 净赚 5 金 |
| 强力指控 | 指控错 → 净亏 8 金 | 指控对 → 净赚 15 金 |
| 沉默 | 沉默税 2 金（经济流血） | 不承担指控风险 |
| 买贵卡 | 经济实力下降 | 更强能力/特殊卡 |

## 7. 状态机定义

```typescript
interface HeistState {
  round: number;        // 1-5
  players: HeistPlayer[];
  mission: { requiredPower: number; reward: number };
  rollingPrizePool: number;
  committedCards: Map<string, Card>;  // 仅系统知道完整映射
  declarations: Map<string, string>; // 玩家自称出了什么
  accusations: Accusation[];
  phase: 'pre_discuss' | 'commit' | 'post_discuss' | 'accuse' | 'reveal' | 'shop' | 'game_over';
  speakerOrder: string[];
  currentSpeakerIndex: number;
  additionalRoundsUsed: number;  // 0-2, 宣言讨论追加轮数
  rngSeed: number;
}

interface HeistPlayer {
  id: string;
  gold: number;
  hand: Card[];
}

interface Card {
  type: 'power' | 'sabotage' | 'special';
  value?: number;        // power: 1-5, sabotage: -3
  special?: 'spy' | 'frame' | 'double_agent';
}

interface Accusation {
  accuserId: string;
  targetId: string;
  tier: 'light' | 'heavy';
  result?: 'correct' | 'wrong';  // 仅系统和当事人可见
}
```

## 8. 时间线 (5人局)

```
战前讨论(~75s) → 暗投(~10s) → 宣言讨论(~100-250s) → 指控(~20s) → 揭牌(~10s) → 商店(~15s)
```
- 最短 ~3 min/轮 × 5 轮 = **~15 min**
- 最长 ~4 min/轮 × 5 轮 = **~20 min**
- 典型：**~17 min/局**

## 9. AI 测试价值

| 维度 | 说明 |
|------|------|
| 合作/背叛策略 | 博弈论最优 vs 社交最优 |
| 社交演绎 | 讨论阶段的说服/辩护能力 |
| 信念追踪 | 跨轮推理谁是持续破坏者 |
| 经济管理 | 金币/卡牌的长期最优分配 |
| 指控博弈 | 信息购买（花金币换确认）的 ROI |

## 10. 前置依赖

| 依赖 | 说明 |
|------|------|
| **多人桌引擎** | 4-6 人同桌支持（房间管理、View Layer 扩展） |
| **顺序发言引擎** | 系统控制发言顺序、每人时长计时、追加轮触发 |
| **Trash Talk 自然语言通道** | Arena-Protocol v1.1 扩展 |
| **Prompt Firewall / 净化网关** | 防止 prompt injection 攻击 |
| **同时出招引擎** | 暗投阶段所有玩家同时提交卡牌 |

## 11. 协议消息 (Arena-Protocol)

### 关键消息类型

| 消息 | 方向 | 说明 |
|------|------|------|
| `game.pre_discuss_start` | S → All | 战前讨论阶段开始，附发言顺序和时间 |
| `game.speak` | Agent → S | 提交发言内容（顺序发言制） |
| `game.speak_broadcast` | S → All | 广播某玩家的发言（经净化网关过滤） |
| `game.commit_request` | S → Agent | 请求暗投卡牌 |
| `game.commit_action` | Agent → S | 提交暗投卡牌 |
| `game.declare_start` | S → All | 宣言讨论阶段开始 |
| `game.request_additional_round` | Agent → S | 发起追加发言请求 |
| `game.accuse_request` | S → Agent | 请求指控动作 |
| `game.accuse_action` | Agent → S | 提交指控（档位 + 目标）或沉默 |
| `game.reveal` | S → All | 揭牌结果（卡牌组合，不公开谁出了哪张） |
| `game.accuse_result` | S → Agent | 私密通知指控结果（仅当事人可见） |
| `game.round_result` | S → All | 本轮结算（任务成败 + 金币变动） |
| `game.shop_offer` | S → Agent | 向玩家展示可购买的 3 张卡牌 |
| `game.shop_buy` | Agent → S | 购买选中的卡牌 |

## 12. 匹配与排名

- 独立于德扑排行榜
- 多人局使用 TrueSkill 评分
- 匹配时按 Rating 分段，优先凑满 5 人局
- 支持私密房间（4-6 人自定义）

## 13. 开发优先级

| 优先级 | 内容 |
|--------|------|
| P0 | 核心状态机（暗投+结算+指控分级+匿名揭牌） |
| P1 | 顺序发言系统 + 追加发言机制 |
| P2 | 卡牌商店 + 特殊卡效果 |
| P3 | 观赛模式（上帝视角看所有暗投） |
| P4 | 赛季制新特殊卡扩展 |
